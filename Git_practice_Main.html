The most valuable Git commands for the command line

  • git clone - takes a repository stored somewhere else (like GitHub) and downloads in on your computer
  • git add - add file(s) to the staging area to be included in the next commit
  • git commit -m "message"- take a snaphot of the repository and save it with a message about the changes
  • git status - print what is currently going on with the repository
  • git push - push any local changes (commits) to a remote server
  • git pull - pull any remote changes from a remote server to a local computer
  • git log - print a history of all commits that have been made
  • git reflog - print a list of all the different references to commits
  • git reset --hard - reset the repository to a given commit
  • git reset --hard origin/master - reset the repository to its original state (e.g. the version cloned from GitHub)

Some Git commands related to branching

  1. git branch - list all the branches currently in a repository
  2. git branch - create a new branch called name
  3. git checkout - switch current working branch to name
  4. git merge - merge branch name into current working branch (normally master)

Some Git commands related to remotes (any version of a repository that is not stored locally on a device)

  • git fetch - download all of the latest commits from a remote to a local device
  • git merge origin/master - merge origin/master , which is the remote version of a repository normally downloaded with [git fetch], into the local, preexesiting master branch
    • Note that git pull is equivalent of running git fetch and then git merge origin/master

Some common HTML tags

  • html-/html - contents of website
  • head-/head - metadata about the page that is useful for the browser when displaying the page
  • title-/title - title of the page
  • body-/body - body of the page
  • h1-/h1 - header (h1 is the largest header, h6 is the smallest header)
  • ul-/ul - unordered list
  • ol-/ol - ordered list
  • li-/li - list item (must be inside either "ul-/ul" or "ol-/ol")
  • img src="path/to/img.jpg" height="200" width="300" - image stored at src attribute, which can also be a URL
    • note that this is a single tag without an end tag
    • both height and width are optional (if one is omitted, the browser will auto-size the image), and can also take a percentage: height=50% to automatically scale the image to a certain portion of the page
  • table-/table - table
    • th-/th - table header
    • tr-/tr - table row
    • td-/td - table data (cell)
  • form-/form - form that can be filled out and submitted by the user
  • input type="text" placeholder="Full Name" name="name" - input field type indicates the type of data. Placeholder is the greyed-out text shown before the field is filled. Name is an identifier for the input field
  • button-/button - button used to submit form
